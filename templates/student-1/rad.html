Brief

Rotorua City Council requires a single-page web application allowing visitors to the town to easily search and locate adventure hotspots for activities such as mountain biking, hiking, and air and water sports using a map-based interface.

Users will be able to map routes from their current or specified location to the hotspot locations, as well as search, categorise, view, and learn more about each destination.

Map Requirements/Use Cases:

  1. Find a place
  2. View that place
  3. Get more details on a place
  4. Navigate to place website
  5. Seach for a place/activity
  6. Show all activities in a category
  7. Find route from “current location“ to place

"It took some time to form a concrete vision of what our apps would be used for, our target audiences, app functionality, and technology for implementation.

"Major decisions were regarding use of the FourSquare Places API or the Google Places API to get data for “places” (both are now utilised in our apps). We also had the choice of using Leaflet, a plugin used previously in class, or Google Maps itself alongside the Google Maps JavaScript API.

"I am very happy with the current application, especially given the numerous problems encountered along the way.

"The whole project was a huge learning curve in terms of JS frameworks, code structuring, API use, security and privacy, time management, collaborative development, the agile approach, scrums, standups, and leadership, and research and testing techniques.
